# InfraGuard Backend

## Overview

The InfraGuard Backend is the high-performance, asynchronous engine powering the Zero-Trust runtime security platform. Built with Python and FastAPI, it acts as a robust middle-layer proxy that intercepts, parses, and evaluates JSON-RPC payloads generated by autonomous AI Agents before they are permitted to execute on host systems. 

It manages the global execution state and utilizes WebSockets to instantly broadcast threat alerts and receive human-in-the-loop decisions from the SOC Dashboard and Mobile Admin applications.

## Architecture

The backend is modularized into several core components:

- **Proxy:** The main entry point for AI Agent communications. It intercepts incoming JSON-RPC traffic and ensures no command reaches the target environment uninspected.
- **Threat Detection Engine:** Analyzes payload structures, command semantics, and arguments against a dynamic ruleset to flag potentially malicious or destructive actions (e.g., unauthorized file deletions, unexpected shell commands).
- **Execution Controller:** Manages the lifecycle of a payload. If a threat is detected, the controller asynchronously pauses the specific execution thread without blocking the main event loop, waiting for an admin decision.
- **State Manager:** Centralized in-memory cache (or database abstraction) that tracks the status of all active, blocked, and quarantined sessions and payloads.
- **FastAPI:** The high-speed ASGI web framework powering the REST and WebSocket endpoints.
- **WebSocket:** Handles bi-directional, sub-millisecond state broadcasts between the State Manager and all connected admin clients (Flutter & Web).
- **JSON-RPC Parser:** Normalizes incoming agent payloads to ensure consistent evaluation by the Threat Detection Engine.

## How to Run

1. **Set up your Python environment (Requires Python 3.10+):**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

2. **Install dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Start the Interactive Agent Simulator & Proxy Engine:**
   *Note: The proxy engine automatically starts the FastAPI server on port 8000 internally. You do not need to run `uvicorn` separately.*
   ```bash
   python simulate_agent.py
   ```

## API Endpoints

- `GET /api/status`
  - **Purpose:** Fetches the current system status and logs.
- `POST /api/resolve`
  - **Purpose:** Receives human-in-the-loop decisions from Admin clients.
  - **Payload:** `{ "incident_id": "...", "action": "ALLOW" | "BLOCK" | "QUARANTINE" }`

## WebSocket Endpoint

- `WS /api/ws`
  - **Purpose:** Persistent connection for Admin clients (Web Dashboard and Flutter).
  - **Action:** Broadcasts real-time state changes, active threats, and execution metrics.

## Agent Simulation

To test the proxy, you run the interactive simulator. This script allows you to choose between various mock AI agents (safe and hacked).

```bash
python simulate_agent.py
```

## Threat Detection Rules

The engine currently evaluates payloads against heuristic rules:
- **High Risk:** Shell commands (`rm -rf`, `wget`, `curl`), unauthorized database drops, and sensitive directory access.
- **Low Risk:** Standard API queries, basic calculations, and benign logging operations.
- **Quarantine Triggers:** Repeated high-risk attempts from a single session ID within a short timeframe.

## Execution Flow

1. AI Agent sends a `POST /api/rpc` payload.
2. The **JSON-RPC Parser** validates the format.
3. The **Threat Detection Engine** evaluates risk.
4. If *Safe*: The **Execution Controller** runs the command and updates the **State Manager**.
5. If *Malicious*: The **Execution Controller** pauses the thread. The **State Manager** pushes an alert via **WebSockets**.
6. Admin clicks ALLOW/BLOCK. A `POST /api/decision` is received.
7. The **State Manager** updates the state, broadcasts the resolution, and releases the paused **Execution Controller** thread.

## Future Scope

- **Machine Learning Integration:** Replace static heuristic rules with an ML model to detect sophisticated zero-day prompt injection obfuscations.
- **Database Persistence:** Migrate the in-memory State Manager to PostgreSQL or Redis for persistent incident timelines and horizontal scaling.
- **gRPC Support:** Expand the proxy to support low-latency gRPC AI agent communications alongside JSON-RPC.
- **Authentication:** Enforce strict JWT validation on the `/api/decision` and WebSocket endpoints.
